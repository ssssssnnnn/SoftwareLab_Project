# سامانه پایش و ارزیابی هوشمند میرورهای نرم‌افزاری (Mirror Monitor)

این پروژه یک سیستم پس‌زمینه (Daemon) و داشبورد تحت وبِ زمان‌واقع (Real-time) است که برای تست، پایش کارایی و سنجش پهنای باند مخازن (Mirrors) پکیج‌های مختلف طراحی شده است. هدف اصلی این ابزار، ارزیابی خودکار زیرساخت‌های شبکه مخازن، محاسبه کمترین تأخیر (Latency) و بیشترین پهنای باند عملیاتی جهت تسریع و بهینه‌سازی فرآیندهای توسعه در لایه دِواپس (DevOps) و مهندسی نرم‌افزار است.

## ویژگی‌های کلیدی
- **پایش دوره‌ای خودکار:** ارزیابی دوره‌ای مخازن در پس‌زمینه با استفاده از APScheduler.
- **تست بلادرنگ و آنی (On-demand):** قابلیت اجرای بنچمارک زنده شبکه با کلیک روی دکمه ابزار.
- **مدیریت پهنای باند هوشمند:** دانلود تکه‌ای (Chunked Stream) فایل‌های واقعی مخازن جهت ارزیابی سرعت بدون مصرف بیش از حد پهنای باند شبکه.
- **داشبورد بهینه‌شده:** رابط کاربری مدرن با استفاده از TailwindCSS مجهز به قفل وضعیت (State Lock) برای جلوگیری از ارسال درخواست‌های تکراری و سربار سرور.
- **لایه تست مستقل و مدرن:** مجهز به تست‌های واحد برای منطق پایتون و توابع فرانت‌اند به صورت کاملاً ایزوله و مستقل (Zero-Package.json برای فرانت‌اند).

## راهنمای نصب و راه‌اندازی گام‌به‌گام
شما می‌توانید این پروژه را به دو روش (محیط بومی پایتون یا کانتینر داکر) راه‌اندازی کنید.

### روش اول: راه‌اندازی با استفاده از ابزار Make (پایتون محلی)
پیش‌نیازها: نصب بودن python3، Node.js/npm و ابزار make.

۱. جهت آماده‌سازی ابزارهای فرانت‌اند (مانند لِینتر و محیط تست)، ابتدا دستور زیر را یک‌بار اجرا کنید:

```Bash
make install-tools
```

۲. جهت آماده‌سازی محیط مجازی، نصب وابستگی‌ها و اجرای مستقیم سرور، دستور زیر را اجرا کنید:

```Bash
make run
```

مرورگر خود را باز کرده و به آدرس زیر مراجعه کنید:
http://localhost:8000

### روش دوم: راه‌اندازی با استفاده از Docker (بسته‌بندی ایزوله)
پیش‌نیازها: نصب و فعال بودن موتور Docker.

۱. برای ساخت ایمیج داکر و اجرای خودکار کانتینر به صورت ایزوله، دستور زیر را وارد کنید:

```Bash
make docker-run
```

۲. وضعیت لاگ‌های فعال کانتینر را می‌توانید با این دستور بررسی کنید:

```Bash
make docker-logs
```
## لایه تست و تضمین کیفیت (Testing & QA)
پروژه دارای یک سیستم تست مستقل برای هر دو بخش بک‌اند و فرانت‌اند است تا پایداری برنامه در شرایط قطعی شبکه تضمین شود.

تست بک‌اند (Pytest): بررسی صحت منطق محاسباتی تابع خلاصه آمار و شبیه‌سازی لبه خطای بحرانی آفلاین بودن تمام میرورها (جلوگیری از ZeroDivisionError).

تست فرانت‌اند (Vitest): تست توابع کمکی فرانت‌اند (مانند تولید خودکار اسکریپت‌های پیکربندی سیستم‌عامل) در یک محیط شبیه‌سازی شده (Mocked Sandbox) با استفاده از ماژول بومی vm در Node.js بدون اعمال سربار به مرورگر واقعی.

برای اجرای تست‌ها از دستورات زیر استفاده کنید:
```Bash
make test          # اجرای همزمان تمامی تست‌های بک‌اند و فرانت‌اند
make test-backend  # اجرای اختصاصی تست‌های پایتون
make test-frontend # اجرای اختصاصی تست‌های جاوااسکریپت با Vitest
```

## دستورات مدیریتی پروژه (CLI Cheatsheet)
* make init: ساخت محیط مجازی .venv و نصب پکیج‌های پایتون به همراه ابزارهای تست.
* make install-tools: نصب سراسری ابزارهای فرانت‌اند (eslint ،prettier و vitest).
* make lint: بررسی خطاها و مشکلات ساختاری کدهای پایتون (Ruff) و جاوااسکریپت (ESLint).
* make format: مرتب‌سازی خودکار کدها بر اساس استانداردهای روز دنیای نرم‌افزار.
* make clean: پاکسازی فایل‌های موقت، پایتون کش (__pycache__) و حذف محیط مجازی.
* make docker-clean: متوقف‌سازی و حذف کامل کانتینر و ایمیج پروژه از داکر.
